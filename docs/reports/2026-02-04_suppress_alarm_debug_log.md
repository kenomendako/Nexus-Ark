# タスク完了レポート: アラーム応答生成時のシステムプロンプト出力廃止

## 概要
アラーム応答生成時に、デバッグモードが強制的に有効になっており、ターミナルに全システムプロンプトが出力される問題を修正しました。

## 実施内容
- `alarm_manager.py` の `trigger_alarm` 関数内で、`debug_mode` を `True` から `False` に変更しました。
- 通常の応答と同様に、明示的なデバッグ時以外はプロンプト内容がターミナルに出力されないようになりました。

## 検証結果
- テストスクリプト `verify_alarm_fix.py` を実行し、`debug_mode` が `False` で AI 実行が呼び出されていることを確認しました。
- ターミナルから不要なデバッグ出力が消えたことを確認しました。

## 影響範囲
- アラーム発火時のターミナルの視認性が向上します。
- AIの応答生成ロジック自体には影響ありません。
